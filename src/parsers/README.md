## Назначение

Модуль содержит набор классов-парсеров, каждый из которых проверяет определённый раздел или аспект отчёта. Все парсеры наследуются от абстрактного базового класса `BaseParser` и реализуют метод `parse()`, возвращающий результаты проверки (нарушения, замечания).

## Состав модуля

- `base_parser.py` – базовый класс `BaseParser`.
- `title_page_parser.py` – проверка титульного листа (наличие ФИО, темы, типа практики, правильное оформление).
- `introduction_parser.py` – проверка введения (наличие целей, задач, актуальности, минимальный объём).
- `sections_parser.py` – общая проверка основных разделов (наличие, порядок, нумерация).
- `formatting_parser.py` – проверка параметров форматирования (шрифт, размер, отступы, поля, интервалы) для всего документа.
- `content_parser.py` – проверка содержания (личные конструкции, объём разделов, шаблонные фразы).
- `conclusion_parser.py` – проверка заключения (наличие выводов, соответствие задачам).
- (опционально) другие парсеры, например, `references_parser.py`, `appendix_parser.py`.

## Входные данные

Каждый парсер получает:
- Данные раздела (например, объект `Section` с заголовком и текстом) или весь документ (для `formatting_parser`).
- Словарь конфигурации с требованиями (например, минимальный объём, допустимые шрифты).

## Выходные данные

Словарь, содержащий результаты проверки. Рекомендуемая структура:
```python
{
    "errors": [{"message": "текст ошибки", "page": 5, ...}],
    "warnings": [{"message": "предупреждение", "page": 3, ...}],
    "info": {...}  # дополнительная информация
}
```

## Базовый класс (BaseParser)
```python
from abc import ABC, abstractmethod

class BaseParser(ABC):
    @abstractmethod
    def parse(self, section_data, config: dict) -> dict:
        """Абстрактный метод, который должны реализовать все наследники."""
        pass
```

## Пример конкретного парсера

## Требования к реализации
Каждый парсер должен быть самодостаточным и тестируемым.

При проверке форматирования (formatting_parser) необходимо использовать данные из RawPDFDocument (координаты, шрифты), которые могут быть доступны через structured_doc.metadata или переданы отдельно.

Сообщения об ошибках должны быть понятны пользователю и содержать номер страницы, если применимо.

Желательно использовать pymorphy3 для анализа текста (личные конструкции, ключевые слова).
