## Назначение

Модуль преобразует сырые данные из `RawPDFDocument` в структурированное представление `StructuredDocument`. На этом этапе:
- текст группируется в логические блоки (заголовки, параграфы);
- определяется тип практики (бакалавр или магистратура) на основе титульного листа;
- сохраняется информация о форматировании (шрифт, размер) для каждого блока.

## Входные данные

- Объект `RawPDFDocument`.

## Выходные данные

- Объект `StructuredDocument` (определён в `src/models/structured_document.py`).

## Основные функции

### `parse(raw_doc: RawPDFDocument) -> StructuredDocument`

Анализирует титульный лист (первая страница), выделяет заголовки по признакам (жирный шрифт, больший размер, центрирование). Создаёт структурированный документ, где каждый раздел содержит заголовок и список параграфов. Также определяет тип практики и сохраняет его в поле `practice_type`.

## Требования к реализации

- Для выделения заголовков можно использовать эвристики: размер шрифта > 14 pt, жирное начертание, выравнивание по центру.
- Для определения типа практики искать на титульном листе слова «бакалавр», «магистр», «бакалавриат», «магистратура» и т.п. (с учётом морфологии через pymorphy3).
- Сохранять исходные координаты для каждого блока (могут пригодиться при проверке форматирования).
- Добавить поле `sections` в `StructuredDocument`, содержащее список объектов `Section`.

## Пример использования

```python
from src.pdf_parser.parser import PDFParser
from src.type_parser.parser import TypeParser

pdf_parser = PDFParser()
raw_doc = pdf_parser.parse("examples/sample_bachelor.pdf")

type_parser = TypeParser()
structured_doc = type_parser.parse(raw_doc)
print(structured_doc.practice_type)  # "bachelor"
print(structured_doc.sections[0].title)  # "Введение"
```
