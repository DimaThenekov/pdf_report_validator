
# Модуль parsers_matcher

**Ответственный**: Ценеков Дмитрий (P3410)

## Назначение

Модуль определяет, какие конкретные парсеры из модуля `parsers` должны быть применены к каждому разделу структурированного документа. Он использует конфигурацию (YAML) и заголовки разделов для сопоставления.

## Входные данные

- Объект `StructuredDocument`.
- Загруженная конфигурация (словарь из YAML).

## Выходные данные

- Список кортежей `(название_раздела, имя_парсера)` или список объектов `(Section, BaseParser)`.

## Основные функции

### `match(doc: StructuredDocument, config: dict) -> List[Tuple[str, str]]`

Анализирует заголовки разделов в `doc` и на основе правил из конфигурации возвращает список соответствий: для каждого раздела указывается, какой класс парсера его обработает. Например, для заголовка «Введение» может быть назначен парсер `IntroductionParser`.

## Требования к реализации

- Правила сопоставления могут быть основаны на:
  - точном совпадении заголовка (из конфига);
  - регулярном выражении;
  - положении раздела (первый, последний).
- Конфигурация может содержать секцию `matching_rules`, где каждому парсеру соответствует список шаблонов заголовков.
- Если для раздела не найдено подходящего парсера, можно применить парсер по умолчанию (`DefaultSectionParser`), который проверяет только базовые требования.
- Важно, чтобы `match` возвращал результат, удобный для дальнейшего вызова парсеров (например, список объектов `(Section, parser_class)`).

## Тестирование

- Тесты находятся в `tests/test_parsers_matcher/`.
- Проверить на разных наборах заголовков (полное соответствие, частичное, отсутствие).
- Проверить загрузку правил из конфигурации.

## Пример использования

```python
from src.parsers_matcher.matcher import ParsersMatcher
from src.parsers import TitlePageParser, IntroductionParser

matcher = ParsersMatcher()
matches = matcher.match(structured_doc, config)
# matches = [("Титульный лист", TitlePageParser), ("Введение", IntroductionParser), ...]
```
