# PDF Report Validator

Система автоматизированной проверки отчётов по производственной практике на соответствие требованиям оформления и содержания.

## Назначение

Проект предназначен для преподавателей и руководителей практики. Позволяет сократить время проверки, унифицировать требования и выявлять формальные нарушения до ручной проверки содержания.

## Основные возможности

- Извлечение текста, шрифтов, координат из PDF (с помощью PyMuPDF).
- Преобразование в структурированное представление (заголовки, параграфы).
- Определение типа практики (бакалавр/магистр) по титульному листу.
- Проверка наличия обязательных разделов, порядка следования, нумерации.
- Проверка форматирования: шрифт, размер, отступы, поля, интервалы.
- Проверка содержания: личные конструкции, минимальный объём разделов.
- Генерация детализированного отчёта (текст, JSON, HTML).

## Архитектура

Система состоит из четырёх основных модулей:

1. **pdf_parser** – извлекает сырые данные из PDF (текст, шрифты, координаты).
2. **type_parser** – преобразует сырые данные в структурированный документ, определяет тип практики.
3. **parsers_matcher** – на основе структуры и конфигурации сопоставляет разделы с конкретными парсерами.
4. **parsers** – набор парсеров, каждый проверяет определённый раздел или аспект документа.

Модули взаимодействуют через общие модели данных (`models/`). Конфигурация загружается из YAML-файлов.

## Технологии

- Python 3.10+
- PyMuPDF (fitz) – работа с PDF
- pymorphy3 / natasha – обработка русского языка
- PyYAML – конфигурация
- pytest – тестирование
- GitHub Actions – CI

## Установка и запуск

1. Клонировать репозиторий:
```bash
git clone https://github.com/your-org/pdf-report-validator.git
cd pdf-report-validator
```

2. Создать виртуальное окружение и установить зависимости:
```bash
python -m venv venv
source venv/bin/activate  # или venv\Scripts\activate на Windows
pip install -r requirements.txt
```

3. Запустить проверку одного файла:
python -m src.main path/to/report.pdf --config config/my_config.yaml

## Тестирование

Запуск всех тестов:
```bash
pytest tests/
```

Для тестирования отдельного модуля:
```bash
pytest tests/test_pdf_parser/
```
